# shortcuts/

Stable slot files for keyboard-shortcut bindings — the answer to Synapse
having no automation API. Each `slot-<set>-<key>.vbs` runs the
[Resolver](../resolver.md) with `--shortcut <set>:<key>`.

**The contract:** these paths NEVER change. A Razer Synapse LAUNCH binding
(including Hypershift) is pointed at a slot file ONCE, by hand, and never
touched again — which preset the slot applies lives in `config.json` and is
freely re-mapped from the GUI/config at any time.

Files are regenerated by `python resolver.py --write-slots` (they only need
regenerating when a shortcut SET or KEY is added/removed, or when the
Python path changes — never on preset re-mapping).

Generated files get no individual docs (project convention).
